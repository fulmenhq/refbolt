**Release date**: 2026-04-02

**16 new providers, incremental sync, user-facing config, and public-readiness.**

## Highlights

- **24 providers total**: +DigitalOcean (6), Cloudflare (4), Mattermost (2), Nextcloud, Stalwart
- **Incremental sync**: Skip unchanged providers via metadata hints — tree SHA for GitHub, HEAD for HTTP. Write-level SHA-256 dedup. `--force` bypass.
- **User-facing config**: `refbolt init` generates config from embedded catalog, `refbolt validate` checks it. Works without cloning the repo.
- **3 splitter formats**: `===` (xAI), `URL:`/`Source:` (Anthropic/DO), YAML frontmatter (Cloudflare)
- **Provider filtering**: `--provider`, `--topic`, `--exclude-provider` flags with union semantics
- **First-run guidance**: Credential hints in `init`, `validate`, generated config, and README

## Fetch Strategy Enhancements

### YAML Frontmatter Splitter (new)

Cloudflare's `llms-full.txt` uses `---/title:/---` YAML frontmatter blocks as section delimiters instead of `URL:` or `Source:` lines. `SplitFrontmatterFullTxt` handles this format, stripping Cloudflare-specific boilerplate (Skip to content, Was this helpful, Edit page, Copy page) to produce clean Markdown.

The splitter chain now tries three formats in order:

1. `===/<path>===` delimiters (xAI)
2. `URL:` / `Source:` lines (Anthropic, DigitalOcean)
3. YAML frontmatter blocks (Cloudflare)

### URL Prefix Filtering

`FilterByBaseURL` scopes split pages by `base_url` prefix. DigitalOcean uses a single 40MB `llms-full.txt` for all documentation; each scoped provider entry archives only sections whose `Source:` URL matches its prefix.

### Configurable Fetch Timeout + Jina Retry

Per-provider `fetch_timeout` (default 30s). Jina fetcher retries once with 2x timeout on deadline exceeded. OpenAI set to 90s.

## Provider Filtering

```bash
refbolt sync --provider openai --provider anthropic   # specific providers
refbolt sync --topic llm-api                           # all in topic
refbolt sync --all --exclude-provider trino            # all except
```

Selection semantics: `--provider` overrides `enabled: false`, `--all`/`--topic` respect it. Error on unknown slugs or empty set.

## Incremental Sync

Per-provider `.sync-meta.json` with config hash, content hash, and strategy-specific hints:

- **github-raw**: tree SHA comparison (one API call skips 600+ file fetches)
- **native/llmstxt-split**: HEAD request compares ETag + Content-Length
- **jina**: always fetches (no server-side hints), write-level dedup only

`--force` bypasses all skip logic.

## User-Facing Configuration (FA-097)

refbolt now ships with an embedded provider catalog and schema, making it usable
without cloning the source repo.

### New commands

- **`refbolt init`**: Generate a `providers.yaml` from the built-in catalog. Select
  topics via `--topic` flags or `--all` for everything. YAML goes to stdout;
  status messages to stderr.
- **`refbolt validate`**: Check a config file against the embedded schema. Reports
  errors (exit 1) and advisory warnings (exit 0) including missing credential
  env vars for Jina/GitHub providers.

### Config resolution

1. `--config <path>` flag
2. `REFBOLT_CONFIG` env var
3. `./providers.yaml` (CWD)
4. `~/.config/refbolt/providers.yaml` (XDG)
5. Built-in catalog (zero-config fallback, `archive_root` defaults to `./archive`)

### `--config` global flag

Available on all commands (`sync`, `validate`, `init`).

## First-Run Credential Guidance (FA-098)

- README.md now has a Prerequisites section with `JINA_API_KEY` and `GITHUB_TOKEN`
- `refbolt init` emits stderr hints when selected providers need API keys
- `refbolt validate` warns when credential env vars are missing for enabled providers
- Generated `providers.yaml` includes inline comments for credential-dependent providers
