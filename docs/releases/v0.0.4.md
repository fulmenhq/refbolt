# refbolt v0.0.4

**Release date**: 2026-04-22

**Operational foundation, provider browsing, and four new providers.**

## Highlights

- **`docker-compose.yml`**: Turnkey scheduled archiving — one-shot CLI, a
  persistent runner (supercronic + git + ssh), and a `--profile git`
  variant for signed commits and pushes. Three services around a host
  bind-mounted `./archive` directory. (FA-012)
- **`refbolt catalog` command**: Browse the embedded provider catalog
  without cloning the repo. `list`, `show <slug>`, and `topics`
  subcommands, enriched with registry metadata (descriptions, page
  counts). Includes JSON envelope, `--topic` / `--strategy` filters,
  and "did you mean?" suggestions. (FA-101)
- **27 providers total**: +Figma REST API (new `design-platform` topic),
  +Hetzner Cloud API / Cloud Docs / Networking Docs (cloud-infra).
  23 → 27 providers, 7 → 8 topics. (FA-104, FA-091)
- **Architecture doc refresh**: `docs/ARCHITECTURE.md` fully rewritten to
  match v0.0.3 reality — real command surface, embedded catalog/registry,
  all five fetch strategies with splitter variants, incremental sync
  (FA-095), date-versioned archive writer. Drops the stale "planned"
  framing and ghcr.io references. (FA-100)
- **DDR-0001 reconciliation + TZ=UTC sweep**: DDR-0001's prose now
  matches `internal/archive/writer.go` exactly (calendar-day keying,
  dedup-then-overwrite semantics). Runner examples and operational
  guidance standardize on `TZ=UTC` with override notes. (FA-105)
- **First-run usability polish** (FA-111): `refbolt --version` now
  works; `--verbose` actually enables debug logging (was dead code);
  every credential warning surfaces a get-a-key URL; `refbolt sync`
  with no selectors now shows concrete next-step hints; duplicate
  `Error:` lines fixed. README gains a "Getting Started (5 minutes)"
  numbered walkthrough.
- **Bundled fixes**: `refbolt init` now emits schema-valid YAML;
  docker-compose correctly honors user config and archive root; catalog
  filtered totals describe the result set, not the full catalog.

### Figma (new topic: `design-platform`)

Spec-only by design. `developers.figma.com` robots.txt explicitly blocks
AI crawlers (`anthropic-ai`, `GPTBot`, `CCBot`); per
[SDR-0001](../decisions/SDR-0001-ethical-fetching-policy.md), refbolt
respects that restriction. The OpenAPI 3.1.0 spec at `figma/rest-api-spec`
(MIT licensed) is the authoritative machine-readable reference and is
what we archive.

### Hetzner (multi-surface, user-configurable)

Hetzner has no first-party source repo or `llms.txt`, so we ship three
slugs under `cloud-infra` — users opt into the surfaces they need,
mirroring the AWS and DigitalOcean multi-guide pattern:

- **`hetzner-cloud-api`**: OpenAPI 3.0.3 spec (JSON) via `github-raw`
  from `MaximilianKoestler/hcloud-openapi`. Upstream is a derivative
  (codegen-friendly, actively maintained, MIT licensed) — not the
  canonical Hetzner-published JSON served via `docs.hetzner.cloud`.
  See `docs/providers/README.md` for the monitoring / exit-criteria
  stance.
- **`hetzner-cloud`**: Jina Reader over `docs.hetzner.com/cloud/` —
  narrative product docs for Cloud.
- **`hetzner-networking`**: Jina Reader over
  `docs.hetzner.com/networking/` — narrative product docs for Networking.

Selection guidance is in `docs/providers/README.md` — short version: if
you're coding against the Cloud API, `hetzner-cloud-api` alone is
usually enough; add `hetzner-cloud` / `hetzner-networking` if you also
want conceptual / operational context.

## `refbolt catalog` Command

`refbolt catalog` is a read-only view over the embedded catalog, joined
by slug with the embedded registry. No filesystem or network dependency.

Quickest way to feel the new command — pick a headline provider and look:

```bash
refbolt catalog show hetzner-cloud-api      # one of the new Hetzner surfaces
refbolt catalog show figma-api              # the new design-platform provider
```

Full command set:

```bash
# Human-readable table (default)
refbolt catalog list

# Filters
refbolt catalog list --topic llm-api
refbolt catalog list --strategy github-raw

# JSON envelope for scripts
refbolt catalog list --json

# Per-provider detail (topic, strategy, sample paths, archive output path)
refbolt catalog show <slug>

# Topic summary with provider counts
refbolt catalog topics
```

The command bypasses normal config loading — `--config` is silently
ignored on `catalog` subcommands for ergonomic consistency, enforced by
a regression test in `internal/cmd/catalog_test.go`.

Registry-only entries (currently `aws-cli`) do not appear in `list` or
`show`; listing a provider that `sync` can't fetch would be misleading.
Disposition of `aws-cli` — promote to catalog or retire from registry —
is tracked as FA-106 backlog.

## docker-compose

```yaml
services:
  refbolt: # one-shot CLI
  runner: # scheduled runner (supercronic + git + ssh)
  runner-git: # profile: git — adds repo/ssh mounts for signed pushes
```

Usage:

```bash
refbolt init --all --output providers.yaml     # prerequisite
docker compose build
docker compose run --rm refbolt sync --all     # one-shot
docker compose up -d runner                    # scheduled
docker compose --profile git up -d runner-git  # with git push
docker compose down                            # archive dir persists
```

Archive storage defaults to a host bind mount (`./archive:/data/archive`)
so the snapshot tree stays directly visible to editors and diff tools.
Named-volume / NFS overrides are documented in
[docs/ARCHITECTURE.md](../ARCHITECTURE.md).

## Doc Refresh (FA-100, FA-105)

`docs/ARCHITECTURE.md` was written at project kickoff and had drifted
significantly through the v0.0.1 → v0.0.3 cycle — it listed commands
that don't exist (`status`, `diff`, `serve`), providers that don't
match the catalog, and an aspirational storage tree that was never
built. The rewrite replaces that with current reality:

- Real CLI surface (`init`, `validate`, `sync`, `version`, `catalog`)
- Correct flag syntax (repeatable `--provider`, not comma-separated)
- All five fetch strategies with their splitter variants
- Incremental sync design (FA-095) with code pointers
- Date-versioned archive writer as actually implemented (DDR-0001)
- Local-binary-first distribution (brew, scoop, GitHub Releases, `go install`)

DDR-0001 was reconciled separately (FA-105): the "each sync creates a
new date directory / files never modified after creation" language is
now aligned with the writer's calendar-day keying and SHA-256 dedup.
Runner examples across ARCHITECTURE.md, development.md, and the crontab
samples all use `TZ=UTC` with override guidance.

## Bundled Fixes

Landed inside feature PRs rather than as standalone briefs, but worth
calling out:

- **`refbolt init` emitted schema-invalid YAML.** Indentation on
  credential-hint comments used a hardcoded 6-space target while yaml.v3
  marshals at 8 spaces, pulling the following provider out of its topic.
  `fetch_timeout` emitted compound Go duration strings (`1m30s`) that
  the schema's single-unit pattern rejects. Switched to `yaml.Node`
  `HeadComment` and single-unit seconds; added regression test in
  `TestInitCmd_RealCatalog_RoundTripsValid`. (landed in PR#30)
- **Compose `runner-git` pointed at tracked catalog, not user config.**
  `REFBOLT_CONFIG` hardcoded `/workspace/configs/providers.yaml`, so
  `docker compose --profile git up -d runner-git` archived the bundled
  catalog instead of the user's `providers.yaml`. Fixed to
  `/workspace/providers.yaml`. (PR#30)
- **Compose missed `REFBOLT_ARCHIVE_ROOT`.** `refbolt init` emits
  `archive_root: ./archive`, which resolved to `/app/archive` inside
  the container — ephemeral, invisible on host. Services now pin
  `REFBOLT_ARCHIVE_ROOT=/data/archive` so writes always land in the
  bind mount. (PR#30)
- **Catalog list filtered totals wrong.** `--topic` / `--strategy`
  filters returned the filtered provider rows but reported
  `topics_total: 7` (full catalog) in the JSON envelope and stderr
  hint. Now both describe the rendered result set. (PR#34)

## First-Run Usability Polish (FA-111)

Delivered as part of PR #38 alongside the release-docs so the very
first external installation lands clean. Every item here came from an
end-to-end walk-through of what a brew/scoop user sees in the first
60 seconds, then sharpened by devrev + cxotech pre-read.

**CLI surface:**

- `refbolt --version` — flag form wired alongside the existing `version`
  subcommand. Previously returned `unknown flag`.
- `--verbose` / `-v` — previously dead code. Now enables gofulmen
  debug-level logging in `config.Load()`, surfacing existing
  `log.Debug(...)` calls.
- **Duplicate `Error:` lines** — every failed command previously
  printed the error twice (cobra + main.go). `rootCmd.SilenceErrors`
  now stops the cobra auto-print; main.go is the single source.
- `refbolt sync` with no selector now emits a multi-line hint block
  naming `--all`, `--topic`, `--provider`, and `refbolt catalog list`.
- Pluralization fixed — `1 provider` (not `1 providers`) everywhere
  the CLI renders counts.

**Credential guidance:**

- Every credential-warning surface (`refbolt init` stderr, `refbolt
validate` warnings, `refbolt catalog show <slug>`) now includes a
  get-a-key URL:
  - `JINA_API_KEY` → `https://jina.ai/reader`
  - `GITHUB_TOKEN` → `https://github.com/settings/tokens`
- `refbolt init` without `--output` now emits a single-line tip about
  using `--output providers.yaml` — teaches the pattern without
  breaking pipe workflows.
- `refbolt validate` on the zero-config path now follows the existing
  `Config source: (embedded catalog)` line with a customization tip
  pointing at `refbolt init --all --output providers.yaml`.

**Documentation:**

- `README.md` — new "Getting Started (5 minutes)" numbered walkthrough
  targeted at brew/scoop installers: browse → zero-config sync →
  init → edit → sync → credentials.

## Upgrade Notes

- **Regenerate any cached `providers.yaml` that was generated
  pre-v0.0.4.** Files produced by `refbolt init` before this release
  fail `refbolt validate` due to the indentation and duration-format
  bugs. Run `refbolt init --all --output providers.yaml --force` to
  regenerate from the fixed emitter.
- **`refbolt sync --all` now picks up Figma automatically** (new
  `design-platform` topic, single `figma-api` provider). If that's
  unwanted, exclude with `--exclude-provider figma-api`.
- **Compose users** who previously overrode `REFBOLT_CONFIG` on the
  runner-git service to use `/workspace/configs/providers.yaml`
  should drop that override — the service now reads the repo-root
  `providers.yaml` by default.
- **Scripts that grep stderr for "Error:" lines** should note that
  failed commands now print exactly one `Error:` line, not two.
  Matching logic that relied on the duplicate will still work (a
  "contains 'Error:'" check is unaffected); tests that counted the
  occurrences need to be updated.
- **`--verbose` / `-v` now actually emits debug output.** Scripts that
  set `-v` expecting silence should drop the flag.
