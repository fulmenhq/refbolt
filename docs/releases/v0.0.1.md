# refbolt v0.0.1

**Release date**: 2026-03-23

**Bolt down the reference docs and ship faster.**

First functional release of refbolt — a container-first CLI for archiving web documentation into clean, date-versioned Markdown trees.

## Highlights

- **8 providers verified**: xAI, Anthropic, Pydantic, OpenAI, Trino, kubectl, AWS Glue, AWS Bedrock
- **5 fetch strategies**: native, jina, auto, github-raw, llmstxt-hierarchical
- **Git automation**: `--git-commit`, `--git-push`, `--git-trailer` for scheduled archive workflows
- **Two container images**: slim CLI (distroless, ~8MB) and scheduled runner (Debian + supercronic + git, ~80MB)
- **CI/CD**: GitHub Actions for CI on push/PR, draft release on `v*` tags with manifest-only signing handoff

## Fetch Strategies

| Strategy               | When Used                                            | Auth                         |
| ---------------------- | ---------------------------------------------------- | ---------------------------- |
| `native`               | Provider serves `.md` or `llms-full.txt` directly    | None needed                  |
| `jina`                 | Provider serves HTML only                            | `JINA_API_KEY` (optional)    |
| `auto`                 | Try native first, fall back to Jina if HTML detected | `JINA_API_KEY` (optional)    |
| `github-raw`           | Docs hosted as Markdown in a GitHub repo             | `GITHUB_TOKEN` (recommended) |
| `llmstxt-hierarchical` | Top-level llms.txt index with per-service endpoints  | None needed                  |

## Git Automation

```bash
refbolt sync --all --git-commit                    # commit after sync
refbolt sync --all --git-commit --git-push         # commit and push
refbolt sync --all --git-commit --git-trailer "Co-Authored-By: ..."  # with attribution
```

Safety: archive-only staging, no force push, no empty commits, pre-existing dirt detection, canonicalized paths.

## Container Images

```bash
# Slim CLI (~8MB, distroless)
make docker-build
docker run --rm refbolt:local version

# Scheduled runner (~80MB, Debian + supercronic + git)
make docker-build-runner
docker run -d \
  -v ./configs/providers.yaml:/work/providers.yaml:ro \
  -v ./archive:/data/archive \
  -v ./crontab:/etc/refbolt/crontab:ro \
  refbolt-runner:local
```

## CI/CD

- **CI**: format-check → build-test on push to main and PRs (`test-short`, no live network)
- **Release**: `v*` tag → VERSION validation → 5-platform cross-build → draft GitHub Release
- **Signing**: manifest-only provenance, operator signs locally, uploads via `make release-upload`
